「echo + Standard Go Project Layout」で、最小CRUDアプリを生成してください。

【アプリ概要】
- 名前: whiscript（apps/whiscript 配下）
- 認証なし
- DB: SQLite + sqlx
- Web: htmx + _hyperscript + Tailwind（すべてCDN、Alpine.jsは使わない）
- テンプレート: html/template
- マイグレーション: goose
- Makefile: dev/build/migrate など

【機能（Project CRUD）】
- Projectの一覧・作成・編集・削除（HTMXで非同期更新）
- 一覧はサーバサイドページネーション（クエリ: page, q, sort）
- HTMXの部分テンプレート（list/table行/フォーム/モーダル）を返す
- 差分置換には Idiomorph を使用（hx-swap="morph"）

【構成】
- Standard Go Project Layout に準拠
- internal/{handler,service,repository,model}
- cmd/whiscript（main）, apps/whiscript（アプリ資材）
- migrations/（goose用）
- web/templates/（html/template; base, partials）
- web/static/（必要最小のJS/CSS。基本はCDN）

【実装要件（抜粋）】
- ルータ: Echo v4
- DB接続は `internal/repository/db.go` に集約
- repositoryはsqlxでCRUD（NamedExec/Select, StructScan）
- serviceはビジネスロジック・入力検証
- handlerはEchoでHTTP, HTMXリクエスト（`HX-Request`検知）に応じて
  - 非HTMX: layout込みのフルHTML
  - HTMX: 部分テンプレート断片を返す（例: `<tbody>…</tbody>` or `<tr id="row-:id">…`）
- HTMXイベント: 成功時は対象要素を `hx-swap="morph"` で差し替え、失敗時はエラー断片返却
- SSEやWebSocketは不要（今回はCRUDのみ）
- CDN読み込み例（layoutに記載）:
  - Tailwind: https://cdn.tailwindcss.com
  - htmx: https://unpkg.com/htmx.org@1.9.12
  - _hyperscript: https://unpkg.com/hyperscript.org@0.9.12
  - idiomorph: https://unpkg.com/idiomorph@0.3.0 （`hx-swap="morph"`用拡張）
- テンプレ名例:
  - `base.html`（`{{ block "content" . }}{{ end }}`）
  - `projects/index.html`（一覧ページ／非HTMX）
  - `projects/_table.html`（`<table>`全体）
  - `projects/_tbody.html`（`<tbody>`部分）
  - `projects/_row.html`（1行）
  - `projects/_form.html`（作成/編集フォーム）
  - `projects/_toast.html`（通知）
- ルート例:
  - GET `/projects`（非HTMX: index, HTMX: `_table` or `_tbody`）
  - POST `/projects`（作成→`_row`を返す）
  - GET `/projects/:id/edit`（`_form`返す）
  - POST `/projects/:id`（更新→`_row`差し替え）
  - DELETE `/projects/:id`（削除→`<tr>`をremoveする断片 or 204＋クライアント側remove）
- Makefile タスク例:
  - `make dev`（`go run ./cmd/whiscript`）
  - `make build`（バイナリ出力）
  - `make migrate-up / migrate-down / migrate-new name=xxx`
- goose初期マイグレーション例（projectsテーブル）
  - id, name, description, created_at, updated_at（インデックス適宜）

【HTMXのHTML要件（例）】
- 検索: `<input id="q" hx-get="/projects" hx-trigger="keyup changed delay:300ms" hx-target="#projects-tbody" hx-push-url="true">`
- ソート: `<a hx-get="/projects?sort=name.asc" hx-target="#projects-tbody" hx-push-url="true">Name</a>`
- 追加: `<form hx-post="/projects" hx-target="#projects-tbody" hx-swap="beforeend morph">…`
- 更新: `<form hx-post="/projects/:id" hx-target="#row-:id" hx-swap="morph">…`
- 削除: `<button hx-delete="/projects/:id" hx-target="#row-:id" hx-swap="outerHTML">削除</button>`
